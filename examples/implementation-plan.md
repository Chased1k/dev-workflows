# The Perfect Cut — Implementation Plan

> Generated by PRD Builder. Agent-executable roadmap with explicit completion criteria.
> Rule: No sprint begins until previous sprint acceptance criteria pass.

## Phases

### Phase 0: Foundation

**Goal:** Monorepo, auth, database schema, shared package, CI
**Human review required:** yes

#### Sprint 0.1: Repository Setup

- **Scope (Included):**
  - Turborepo + pnpm workspaces monorepo
  - Next.js 14 app scaffold (`apps/web`)
  - Shared package scaffold (`packages/shared`)
  - Tailwind CSS + shadcn/ui setup
  - ESLint + Prettier config
  - Vitest setup
  - TypeScript strict mode
- **Excluded:**
  - Database schema
  - Auth
  - Any business logic
  - Any UI beyond scaffold
- **Acceptance criteria:**
  - [ ] `pnpm install` succeeds
  - [ ] `pnpm dev` starts Next.js dev server
  - [ ] `pnpm test` runs (even if 0 tests)
  - [ ] `pnpm typecheck` passes
  - [ ] `pnpm lint` passes
- **Files to create/modify:**
  - `package.json`, `pnpm-workspace.yaml`, `turbo.json`
  - `apps/web/` — Next.js scaffold
  - `packages/shared/` — package scaffold
  - Config files: tsconfig, eslint, prettier, vitest
- **Dependencies:** none

#### Sprint 0.2: Supabase + Auth

- **Scope (Included):**
  - Supabase project setup
  - Auth (login, signup, password reset)
  - Server client (request-scoped)
  - Browser client
  - Auth middleware
  - Login/signup pages
- **Excluded:**
  - Database tables beyond auth
  - Athlete profiles
  - RLS policies
- **Acceptance criteria:**
  - [ ] User can sign up
  - [ ] User can log in
  - [ ] User can log out
  - [ ] Protected routes redirect unauthenticated users
  - [ ] Server client works in API routes
- **Files to create/modify:**
  - `apps/web/lib/supabase/server.ts`
  - `apps/web/lib/supabase/client.ts`
  - `apps/web/app/(auth)/login/`
  - `apps/web/app/(auth)/signup/`
  - Auth middleware
- **Dependencies:** Sprint 0.1

#### Sprint 0.3: Database Schema + Migrations

- **Scope (Included):**
  - All Phase 1 tables with Phase 2 columns (organization_id, coach_id)
  - RLS policies
  - Migration files
  - Seed data for development
- **Excluded:**
  - Query functions
  - API endpoints
  - UI that reads from DB
- **Acceptance criteria:**
  - [ ] All tables exist in Supabase
  - [ ] RLS policies prevent unauthorized access
  - [ ] Migrations apply cleanly via `supabase db push`
  - [ ] Seed data inserts without errors
- **Files to create/modify:**
  - `supabase/migrations/` — all migration files
  - `supabase/seed.sql`
- **Dependencies:** Sprint 0.2

**Goal:** AI coach chat, education library, rehydration protocol

  - AI call log table
  - Tests for router and context builder
- **Excluded:**
  - Chat UI
  - Session summaries
